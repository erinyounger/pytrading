# 功能规范: 股票关注列表

**功能分支**: `002-stock-watchlist`
**创建时间**: 2026-03-07
**状态**: 草稿
**输入**: 用户描述: "对回测出来的股票进行人工挑选关注，加入到关注列表，加入列表中的股票独立计算收益率、胜率等，用于挑选一批当前策略下收益率高、策略比较适用的股票进行跟踪交易。挑选出来的股票可能是趋势即将开始，可能需要每天例行回测，回测后进入趋势时改变状态并高亮提醒，指导用户交易。"

## 用户场景与测试 *(必填)*

### 用户故事 1 - 从回测结果关注股票 (优先级: P1)

用户在回测结果列表或仪表盘推荐列表中发现有潜力的股票，点击"关注"按钮将其加入关注列表。关注时系统自动记录当前回测所用的策略ID，系统记录该股票的关注信息。

**优先级原因**: 这是关注列表的入口操作，是所有后续功能的基础。没有加入关注的能力，其他功能均无法运作。

**独立测试**: 可以通过在回测结果页面点击关注按钮，验证股票是否出现在关注列表页面中来完全测试。交付的价值是用户能够手动管理自己感兴趣的股票集合。

**验收场景**:

1. **给定** 用户在回测结果列表中看到一支股票, **当** 点击该行的"关注"按钮时, **那么** 该股票被加入关注列表，按钮变为"已关注"状态
2. **给定** 用户已关注某支股票, **当** 在回测结果列表再次查看时, **那么** 该股票的"关注"按钮显示为"已关注"
3. **给定** 用户在关注列表中查看已关注的股票, **当** 点击"取消关注"按钮时, **那么** 该股票从关注列表中移除

---

### 用户故事 2 - 查看关注列表及独立指标 (优先级: P1)

用户进入独立的"关注列表"页面，以表格形式查看所有关注的股票。每支股票显示最新一次回测的独立指标：收益率、胜率、夏普比率、最大回撤、当前关注类型等。支持按指标排序和筛选。

**优先级原因**: 这是用户日常使用的核心页面。关注列表需要提供清晰的总览，让用户快速判断哪些股票值得关注交易。

**独立测试**: 可以通过在侧边栏导航进入关注列表页面，检查已关注股票是否显示正确的指标数据来测试。交付的价值是用户拥有一个集中管理和监控关注股票的独立界面。

**验收场景**:

1. **给定** 用户已关注了 5 支股票, **当** 进入关注列表页面时, **那么** 能看到 5 支股票的完整指标（收益率、胜率、夏普比率、最大回撤、关注类型）
2. **给定** 关注列表中有多支股票, **当** 按收益率降序排序时, **那么** 列表按收益率从高到低排列
3. **给定** 关注列表中有股票, **当** 点击某支股票时, **那么** 弹出K线图及详细回测数据

---

### 用户故事 3 - 每日自动回测关注股票 (优先级: P2)

系统每天在设定时间对关注列表中的全部股票执行自动回测（使用关注时选定的策略），回测完成后更新各项指标。用户无需手动发起回测即可获取最新数据。

**优先级原因**: 自动回测使关注列表的数据保持时效性，是"趋势高亮提醒"功能的前置依赖。但 MVP 阶段可暂用手动触发替代。

**独立测试**: 可以通过手动触发"立即回测"按钮，验证回测完成后关注列表的指标是否更新来测试。交付的价值是关注列表指标自动保持最新，减少用户手动操作。

**验收场景**:

1. **给定** 关注列表中有 3 支股票, **当** 用户点击"立即回测"按钮时, **那么** 系统对这 3 支股票执行回测并更新指标
2. **给定** 系统已配置自动回测时间, **当** 到达设定时间时, **那么** 系统自动对关注列表中的所有股票执行回测
3. **给定** 回测正在进行中, **当** 用户查看关注列表时, **那么** 显示回测进度

---

### 用户故事 4 - 趋势状态变化高亮提醒 (优先级: P2)

当关注列表中的股票在最新回测后关注类型发生变化（如从"关注中"变为"趋势上涨"），系统在关注列表中对该股票进行高亮显示，提醒用户关注交易时机。"无状态"不参与变化检测。

**优先级原因**: 关注类型变化是用户决策交易的核心信号。高亮提醒帮助用户从关注列表中快速识别需要行动的股票。

**独立测试**: 可以通过对比回测前后的关注类型字段，验证关注类型变化的股票是否被高亮显示来测试。交付的价值是用户不必逐个查看每支股票，系统主动告知哪些股票值得立即关注。

**验收场景**:

1. **给定** 关注的某支股票关注类型从"关注中"变为"趋势上涨", **当** 用户打开关注列表时, **那么** 该股票行以醒目的高亮样式显示，并标注关注类型变化方向
2. **给定** 多支股票关注类型发生变化, **当** 用户打开关注列表时, **那么** 变化的股票排在列表顶部
3. **给定** 用户已查看过关注类型变化的股票, **当** 用户点击"已读"或查看详情后, **那么** 高亮状态消除

---

### 边界情况

- 当用户关注的股票被退市或停牌时，系统在关注列表中标注"停牌/退市"状态，不执行回测
- 当关注列表为空时，页面显示引导提示，引导用户去回测结果页面添加关注
- 当同一支股票在不同策略下都被关注时，作为独立的关注条目分别展示
- 当自动回测执行失败时（如网络异常），记录错误并在下次重试，关注列表显示"回测失败"状态
- 当用户关注了大量股票（>50支）时，自动回测需要队列化执行，避免资源耗尽
- 当回测正在进行中时，"立即回测"按钮禁用并显示回测进度，回测完成后恢复可点击

## 需求 *(必填)*

### 功能需求

- **FR-001**: 用户必须能够从回测结果页面和仪表盘推荐列表中，将股票加入关注列表
- **FR-002**: 用户必须能够从关注列表中移除（取消关注）股票
- **FR-003**: 系统必须提供独立的"关注列表"页面，作为侧边栏导航的一级菜单项
- **FR-004**: 关注列表必须展示每支股票最新一次回测的关键指标：收益率、胜率、夏普比率、最大回撤、关注类型、当前价格
- **FR-005**: 关注列表必须支持按指标排序（收益率、胜率、夏普比率）和按关注类型筛选
- **FR-006**: 用户必须能够在关注列表页面查看股票的K线图和交易记录
- **FR-007**: 用户必须能够手动触发"立即回测"，对关注列表中的全部股票执行回测并更新指标
- **FR-008**: 系统必须在关注股票的关注类型发生变化时，通过行高亮和关注类型变化标签提醒用户
- **FR-009**: 关注类型变化的股票必须默认排在列表顶部，便于用户快速识别
- **FR-010**: 系统必须记录每次回测的关注类型，以便对比前后变化
- **FR-011**: 关注列表必须持久化存储，跨会话保留
- **FR-012**: 回测结果页面和仪表盘中已关注的股票必须显示"已关注"标记

### 关键实体

- **关注条目 (WatchlistItem)**: 代表用户关注的一支股票。关键属性：关联的股票代码、股票名称、关联策略ID、关注时间、最新回测指标快照（收益率、胜率、夏普比率、最大回撤、关注类型）、上一次关注类型（用于变化对比）、最近回测时间。无中间状态，只有"关注"和"取消关注"两种操作
- **回测指标快照**: 每次回测后写入关注条目的指标数据，包含收益率、胜率、夏普比率、最大回撤、关注类型、回测开始时间、回测结束时间
- **关注类型**: 关注列表中股票的状态分类，共 5 种：
  - **无状态** ← TrendingType: Unknown、UpDown（无明确趋势方向）
  - **关注中** ← TrendingType: Observing（出现上涨信号，等待确认）
  - **趋势上涨** ← TrendingType: RisingUp、ZeroAxisUp（策略识别到上涨趋势）
  - **趋势下行** ← TrendingType: DeadXDown、FallingDown（策略识别到下行趋势）
  - **趋势结束** ← 由清仓信号触发（非 TrendingType 映射），表示趋势终结、仓位已清

## 成功标准 *(必填)*

### 可衡量的结果

- **SC-001**: 用户可以在 3 秒内完成"关注"操作（从点击按钮到看到确认反馈）
- **SC-002**: 关注列表页面在 2 秒内加载完毕并显示全部关注股票及其指标
- **SC-003**: 手动触发回测后，50 支关注股票的回测在 10 分钟内全部完成并更新指标
- **SC-004**: 关注类型变化的股票在关注列表中 100% 被高亮标注，无遗漏
- **SC-005**: 用户能够在关注列表页面完成 90% 的日常决策操作（查看指标、查看K线、识别关注类型变化），无需跳转到其他页面

## 假设

- 关注列表为单用户场景（当前系统无多用户/鉴权机制），无需考虑用户权限
- 自动回测（定时触发）在 P2 阶段实现，MVP 阶段提供手动触发"立即回测"按钮即可
- 关注列表中每支股票绑定一个策略，同一股票可在不同策略下分别关注
- 趋势变化的判定依据为：当前关注类型与上一次记录的关注类型不一致
- 关注条目仅保存策略ID，回测时间范围始终使用"最近N个交易日"动态滚动，确保每次回测使用最新数据

## 澄清记录

### Session 2026-03-07

- Q: 关注列表中的趋势状态分类有哪些？ → A: 采用"关注类型"术语，5 种状态：无状态（Unknown/UpDown）、关注中（Observing）、趋势上涨（RisingUp/ZeroAxisUp）、趋势下行（DeadXDown/FallingDown）、趋势结束（清仓信号触发）
- Q: 关注股票时需要保存哪些回测配置用于后续重复回测？ → A: 仅保存策略ID，回测时间范围始终用"最近N个交易日"动态滚动
- Q: 关注条目的"暂停"状态是如何触发的？ → A: 去掉暂停状态，只有"关注"和"取消关注"两种操作
- Q: 用户在回测进行中再次点击"立即回测"时应如何处理？ → A: 回测中禁用按钮，显示进度，完成后恢复可点击
- Q: 内部 TrendingType 到关注类型的映射关系是什么？ → A: Observing→关注中, RisingUp/ZeroAxisUp→趋势上涨, DeadXDown/FallingDown→趋势下行, 清仓信号→趋势结束, Unknown/UpDown→无状态
